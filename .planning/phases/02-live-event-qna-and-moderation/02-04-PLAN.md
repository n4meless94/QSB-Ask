<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public browser to submit action | Anonymous participant submits untrusted text |
| submit action to participant session | Cookie token must prove event session ownership |
| public read helper to browser | Only sanitized approved question rows may cross |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04-01 | Information Disclosure | `listPublicQuestions` | mitigate | Query only `live`/`answered` rows and safe columns; tests reject pending leakage. |
| T-02-04-02 | Spoofing | `submitParticipantQuestion` | mitigate | Validate raw participant token hash for the same event before insert. |
| T-02-04-03 | Tampering | question text/rules | mitigate | Server-side length/rate/duplicate validation before insert. |
| T-02-04-04 | Denial of Service | repeated submissions | mitigate | Enforce event `question_rate_limit_seconds` per participant session. |
</threat_model>
