<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser/client code to Supabase | Untrusted client requests may try to read or mutate records directly. |
| Next.js server to Supabase service role | Privileged key can bypass RLS if misused. |
| participant/public access to event data | Future public reads must never leak pending or archived moderation data. |
