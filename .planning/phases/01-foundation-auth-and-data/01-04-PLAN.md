<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| organiser browser to event server action | Event form fields are untrusted input. |
| server action to Supabase tables | Event creation mutates protected tables and memberships. |
| clipboard interaction | Browser clipboard API may fail or be unavailable. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04-01 | T | `src/lib/events/validation.ts` | mitigate | Validate all event fields server-side before Supabase mutation. |
| T-01-04-02 | E | `src/lib/events/events.ts` | mitigate | Scope list/create operations to authenticated user and rely on RLS membership policies. |
| T-01-04-03 | I | dashboard event list | mitigate | Query only accessible events and never expose events outside the user's active membership. |
| T-01-04-04 | R | event creation | mitigate | Set `created_by` and active organiser membership on creation for auditability. |
| T-01-04-05 | D | create event action | mitigate | Prevent duplicate submits in UI and make server action idempotency-safe enough to avoid accidental double creation from one submit. |
</threat_model>
