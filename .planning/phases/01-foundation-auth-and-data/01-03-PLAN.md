<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser form to server action | Email, password, reset token, and password fields are untrusted. |
| server action to Supabase Auth | Authentication decisions cross into managed Supabase Auth. |
| middleware to protected routes | Session state determines access to organiser surfaces. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03-01 | S | `src/app/(auth)/actions.ts` | mitigate | Authenticate only through Supabase Auth and validate form input server-side. |
| T-01-03-02 | D | `src/lib/auth/lockout.ts` | mitigate | Enforce five failures within 15 minutes causing 30-minute lockout before calling Supabase sign-in. |
| T-01-03-03 | I | password reset request | mitigate | Use non-enumerating confirmation copy for all accepted reset requests. |
| T-01-03-04 | E | `src/middleware.ts` | mitigate | Redirect unauthenticated or inactive users away from `(app)` routes before rendering protected data. |
| T-01-03-05 | R | `login_attempts` | mitigate | Persist login attempts with timestamp and success flag for auditability. |
</threat_model>
