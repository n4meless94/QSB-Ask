<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| organiser browser -> deck server actions | Untrusted upload metadata and file content cross into server validation. |
| deck helpers -> Supabase storage/database | Role-checked application code writes event-scoped deck records and storage paths. |
| staff presenter access -> deck metadata | Speaker/moderator access may load safe deck metadata but must not mutate deck state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01-01 | Elevation of Privilege | src/lib/slides/decks.ts | mitigate | Require organiser role before upload, replace, or remove helpers. |
| T-05-01-02 | Tampering | src/lib/slides/validation.ts | mitigate | Validate MIME/type, extension, size, and page limit server-side before accepting deck metadata. |
| T-05-01-03 | Information Disclosure | src/app/(app)/events/[eventId]/page.tsx | mitigate | Render deck mutation controls only for organisers and never expose storage credentials or raw signed URLs. |
| T-05-01-04 | Denial of Service | slide deck upload | mitigate | Enforce upload size/page limits and reject unexpected content early. |
</threat_model>
