# QSB Ask Coolify Deployment Runbook

**Status:** v1 pilot readiness
**Target URL:** `https://ask.qsbportal.com.my`
**Health path:** `/api/health`

## Deployment Boundary

QSB Ask must run as a Coolify-managed Next.js application resource on QSB VPS. Do not create an unmanaged long-lived Docker Compose service or ad hoc VPS process for v1.

Managed Supabase remains the approved v1 backend for Auth, Postgres, Realtime, and data storage.

## Coolify Resource Setup

1. Create or update a Coolify application resource for this repository.
2. Use Dockerfile build mode for the included standalone Next.js container, or keep Nixpacks only if QSB operations explicitly chooses that path.
3. Set exposed port to `3000`.
4. Configure health checks in Coolify UI:
   - Path: `/api/health`
   - Expected response: `200`
   - Health checks enabled before routing production traffic.
5. Confirm Coolify/Traefik routes HTTPS traffic to the healthy application container.

The Dockerfile follows the official Next.js standalone pattern: it sets `QSB_ASK_STANDALONE_OUTPUT=1`, `next.config.ts` enables `output: "standalone"` for that build, static assets are copied into the runtime image, and `server.js` runs with `HOSTNAME=0.0.0.0` and `PORT=3000`.

## Runtime Environment Variables

Configure these in Coolify as runtime variables. Do not paste values into repository files or logs.

| Variable | Production value expectation |
|----------|------------------------------|
| `NEXT_PUBLIC_SUPABASE_URL` | Managed Supabase project URL. |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase anon key. Public but still managed through Coolify. |
| `SUPABASE_SERVICE_ROLE_KEY` | Server-only Supabase service role key. Keep runtime-only where possible. |
| `NEXT_PUBLIC_SITE_URL` | `https://ask.qsbportal.com.my` |
| `APP_JOIN_URL_BASE` | `https://ask.qsbportal.com.my/join` |
| `APP_SESSION_IDLE_TIMEOUT_SECONDS` | `28800` unless QSB changes the pilot timeout. |

## DNS And Domain Cutover

1. Confirm `ask.qsbportal.com.my` DNS points to the QSB VPS / Coolify ingress target used by operations.
2. Add `https://ask.qsbportal.com.my` to the Coolify application domain settings.
3. Let Coolify/Traefik provision HTTPS.
4. Before announcing the pilot URL, verify:
   - `curl -i https://ask.qsbportal.com.my/api/health` returns HTTP 200.
   - The JSON response has `"ok": true`.
   - No secret values appear in the response.
   - Public join links generated by the app use `https://ask.qsbportal.com.my/join`.

## Pre-Deploy Checks

Run locally before building the Coolify deployment:

```bash
npm run lint
npm test
npx tsc --noEmit
npm run build
```

To test the standalone output path outside Docker, run:

```bash
QSB_ASK_STANDALONE_OUTPUT=1 npm run build
```

On Windows OneDrive-backed checkouts, standalone file tracing can fail during the final copy into `.next/standalone`; use Docker/Coolify or a non-synced local checkout to verify the standalone packaging path if that occurs.

Optional focused smoke:

```bash
npm run test:e2e -- tests/e2e/foundation.spec.ts
```

## Post-Deploy Smoke

Run these after Coolify reports the container healthy:

1. Open `https://ask.qsbportal.com.my`.
2. Request `https://ask.qsbportal.com.my/api/health`.
3. Sign in as an organiser test account.
4. Create a pilot event and copy the join link.
5. Join from a separate browser/mobile device.
6. Submit one moderated question and approve it.
7. Confirm presenter/audience views update.
8. Publish a survey, submit one response, and confirm presentation results update.
9. Download CSV exports as organiser.

## Rollback

If health checks fail or pilot smoke finds a blocker:

1. Keep the previous Coolify deployment available if rollback is enabled.
2. Disable production routing to the unhealthy deployment.
3. Record failed health output, Coolify deployment log excerpt, commit hash, and environment variable names involved.
4. Do not print or copy secret values into issue notes or chat.

## Source Notes

- Next.js standalone output copies traced runtime files into `.next/standalone` and runs `server.js`.
- Official Next.js self-hosting guidance expects a reverse proxy in front of the app; Coolify/Traefik fills that role for QSB VPS.
- Coolify health checks can gate Traefik routing to healthy resources.
- Coolify environment variables can be scoped to build/runtime; keep server secrets runtime-only where practical.
